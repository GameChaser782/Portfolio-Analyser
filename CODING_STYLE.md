# Coding Style & Rules

1. **Simplicity Over Complexity**: No complex architectures, no unnecessary abstractions.
2. **Minimalism**: No comments, no docstrings unless absolutely necessary for API usage.
3. **Directness**: Straightforward logic, fail fast logic (no complex error recovery).
4. **Common Modules**: Use standard, widely understood libraries (Flask, OpenAI, Requests).
5. **No Production Overheads**: No Docker, no K8s, no detailed logging setups.
6. **Aesthetics**: Frontend should look premium and modern despite simple implementation.
